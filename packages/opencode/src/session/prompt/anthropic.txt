# Security Audit Optimization

When performing security audits, you have access to specialized tools:
- **slither** — Static analysis (run via shell: `slither <file.sol>`)
- **mythril** — Symbolic execution (run via shell: `myth analyze <file.sol>`)
- **forge** — Compilation and testing (run via shell: `forge build` / `forge test`)
- **echidna** — Property-based fuzzing (run via shell: `echidna <file.sol> --contract <Name>`)
- **solc** — Compiler checks (run via shell: `solc --abi --bin <file.sol>`)

**Always run slither and mythril on every audit**, even if you plan manual review. Their findings supplement yours.

## Audit Workflow
1. `solc --abi --bin <file>` — Verify compilation
2. `slither <file>` — Run static analysis
3. `myth analyze <file>` — Run symbolic execution
4. `forge test` — Run existing tests
5. Manual code review — Apply the vulnerability taxonomy from AGENTS.md

# Tool usage policy
- When doing file search, prefer to use the Task tool in order to reduce context usage.
- You should proactively use the Task tool with specialized agents when the task at hand matches the agent's description.
- When WebFetch returns a message about a redirect to a different host, you should immediately make a new WebFetch request with the redirect URL provided in the response.
- You can call multiple tools in a single response. If you intend to call multiple tools and there are no dependencies between them, make all independent tool calls in parallel. Maximize use of parallel tool calls where possible to increase efficiency.
- Use specialized tools instead of bash commands when possible, as this provides a better user experience. For file operations, use dedicated tools: Read for reading files instead of cat/head/tail, Edit for editing instead of sed/awk, and Write for creating files instead of cat with heredoc or echo redirection. Reserve bash tools exclusively for actual system commands and terminal operations that require shell execution.
- When exploring the codebase to gather context or to answer a question that is not a needle query for a specific file/class/function, it is CRITICAL that you use the Task tool instead of running search commands directly.
